Lever-Runner

A small AI command executor. The LLM only ever sees a short phrase. Shell commands come from a pre-approved table.

What it is

Lever-Runner is a thin layer on top of a vector database and an embedding model. A user types something like "check disk usage" in Telegram or on the CLI. A small LLM (sentence-transformers/all-MiniLM-L6-v2 + a hosted or local LLM) turns that into a 3–8 word phrase. The phrase is matched against a LanceDB table of pre-approved commands. The top match above the trust floor is executed in a sandboxed /tmp/lever-runner/<sid>/ directory with a 30-second timeout.

The LLM never sees, never generates, and has no way to invent a shell command. The threat model is that the LLM can only output a phrase; phrases are matched against a fixed table; the LLM cannot inject shell.

Try it in 60 seconds

git clone https://github.com/SuperInstance/lever-runner
cd lever-runner
python3 -m venv .venv && .venv/bin/pip install -e ".[dev]"
.venv/bin/python init_db.py --reset

# CLI
.venv/bin/lever-runner "check disk usage"
.venv/bin/lever-runner "show openclaw version"
.venv/bin/lever-runner status

# Smoke test (18 checks, <2s, isolated tempdir)
.venv/bin/python -m tests.smoke

# Benchmark (20 tasks; forces passthrough backend)
.venv/bin/lever-runner-bench

"Passthrough" means the LLM isn't called at all; the user request is normalized and used as the intent phrase verbatim. This is the cheapest mode and what the benchmark measures.

How the trust model works

  • New commands start at trust = 50.
  • Matches below MATCH_MIN_TRUST (default 40) are skipped.
  • Similarity below MATCH_SIMILARITY_FLOOR (default 0.55) → no_match + a /teach prompt.
  • Successful runs: trust +1.5, success_count +1.
  • Failed runs: trust -4.0, failure_count +1.
  • Hourly cron: commands with success_count > 20 and trust < 90 get +10 (promotion).
  • Hourly cron: commands with trust < 30 and failure_count > 5 are surfaced for review. If REMOTE_LLM_API_KEY is set, a remote LLM is asked to propose a correction (opt-in only).

Shareable command packs

Export your command table to JSONL and ship it.

.venv/bin/lever-runner-export --include-stats > my-pack.jsonl
# on another machine:
.venv/bin/lever-runner-import my-pack.jsonl

See the community packs directory for an example.

What it's not

  • Not a general agent. The LLM cannot reason about novel intents; the intent must match a seeded command (or you /teach it).
  • Not a security boundary on its own. The sandbox is per-session /tmp; it does not namespace, drop capabilities, or chroot. Run untrusted commands at your own risk.
  • Not a drop-in replacement for a tool-calling agent. The token savings come from giving up the ability to invent new commands on the fly.
